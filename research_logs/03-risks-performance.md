# 风险与性能瓶颈调研

## 高优先级风险（摘要）
1. 构建链外网依赖高：release 同步前置在 build，网络抖动会放大失败概率。
2. CI 门禁薄弱：缺少仓库级 workflow，文档包 `test` 仍是占位脚本。
3. 构建扩展性风险：`transformPageData` 逐页执行 git log，页面增长时构建时长可能上升。
4. 搜索扩展性风险：本地搜索索引随内容体量增长，首交互性能可能下降。
5. i18n 漂移风险：已配置中英文，但英文内容覆盖不足，导航与内容长期偏差风险高。

## 建议 Guardrails（可量化）
- `docs build` P95 <= 180s。
- 搜索弹窗首开 P95 <= 300ms（主入口页抽样）。
- PR 必过：docs build、链接检查、i18n 映射检查、核心测试。
- 新增中文页后 48h 内必须有英文占位页（或显式状态标记）。

## 建议缓解策略
- 将 release 同步改为缓存优先，并提供手动刷新脚本。
- 引入最小 CI（构建 + 链接 + i18n + 测试）并设为保护分支必过。
- 对编辑者注入逻辑增加缓存/开关，避免大规模页面构建放大成本。
