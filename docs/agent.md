# Agent 使用说明

本文件说明本项目中 agent 的目的、使用方法和示例，方便开发者和运维团队快速上手。

## 简介

Agent（代理）是自动化执行特定任务的组件，例如：定时任务、自动化部署、内容发布或与第三方服务交互。

## 常见用途

- 自动构建与部署文档到 Vercel。 
- 处理静态资源（例如图片优化、重命名、校验链接）。
- 自动化发布、生成变更日志、或触发 CI 流水线。

## 使用示例

1. 本地调试：在项目目录运行相关脚本（示例）

```bash
# 示例：运行文档构建检查
cd docs
npm run build
```

2. 将 agent 与 GitHub/GitLab webhook 配合，实现自动化触发。

## 维护与贡献

- 新增 agent 时，请在此文档补充用途、入口脚本与运行方式。
- 如果 agent 会修改仓库文件，请在提交前在本地充分测试并包含回滚说明。
