# DeepStudent 核心功能指南（基于主项目代码校准）

> 本页聚焦「用户可感知能力」，描述依据为 `deep-student` 当前前后端实现，而非历史宣发口径。

## 1. 智能对话 (Chat V2)：统一对话入口

智能对话是主交互层。用户从对话开始，按任务逐步调用检索、阅读、制卡和外部工具。

核心体验：
- 多会话管理与流式回复
- 引用学习资源作为上下文
- 研究任务（Todo）与工具调用过程可视化
- 附件输入（文档/图片）与多模态问答

## 2. 学习资源中心 (Learning Hub)

学习资源中心不只是文件列表，而是资源访问与上下文注入中心。

核心体验：
- 访达式浏览与拖拽到对话
- 资源创建、整理、批量操作
- OCR / 向量化状态可追踪
- 与对话、制卡任务、检索链路打通

## 3. Skills：能力按需编排

Skills 把“模型能力 + 工具能力”按任务组合，而不是一次性塞入全部工具。

核心体验：
- 指令型技能（如导师、调研、制卡）
- 工具组技能（检索、笔记、记忆、任务、工作区等）
- 内置 / 全局 / 项目级三级加载
- 可创建、导入、编辑自定义技能

## 4. MCP：外部能力扩展层

DeepStudent 支持 MCP 协议，可将外部服务接入到对话工作流。

核心体验：
- 支持 SSE / WebSocket / HTTP 传输
- 统一管理外部服务配置
- 工具调用结果回流到对话与文档链路
- 可与内置技能组合成完整任务流

## 5. CardForge：任务化制卡流程

制卡是一个“可追踪任务”而非一次性动作。

核心体验：
- 从对话或资源内容发起制卡
- 模板管理与卡片预览
- 任务进度、失败重试、结果导出
- 支持 AnkiConnect 同步与 APKG 导出

## 6. 本地优先数据闭环

所有学习资产最终进入同一数据底座，支持跨模块复用。

核心体验：
- VFS 统一资源表达
- SQLite + LanceDB + Blob 本地存储
- 资源可被检索、引用、再加工
- 备份、恢复、同步与审计能力

## 7. 建议上手路径

1. 完成 API 与模型配置（可先使用推荐模型服务）
2. 在学习资源中心导入一份真实学习资料
3. 回到智能对话通过引用方式发起问答/总结
4. 用技能切换任务模式（调研、制卡等）
5. 在系统设置中配置搜索、备份和同步策略
